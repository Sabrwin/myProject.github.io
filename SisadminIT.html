Лекция о профессии системного администратора

Кто такой системный администратор?

Системный администратор - это специалист, отвечающий за установку, настройку и поддержку компьютерных систем, серверов и сетевой инфраструктуры в организации. Он обеспечивает бесперебойную работу всех IT-систем и решает технические проблемы пользователей.

Системный администратор — это специалист, который занимается настройкой, совершенствованием и поддержкой ИТ-инфраструктуры компании, включая оборудование, периферию, программное обеспечение и сетевые подключения. Правда, очень формальное определение?

То, чем занимается системный администратор, зависит от размера компании, сферы деятельности, опыта и навыков самого админа. Вместо того чтобы давать определение, лучше выделим конкретные типы сисадминов.

  • Эникей — начинающий системный администратор, который выполняет базовые функции по настройке оборудования и программного обеспечения. Обычно помощник старшего сисадмина или админ в небольшой неайтишной компании, который закрывает текущие инциденты.
  • Системный администратор (он же труъ админ) — специалист широкого профиля, который отвечает за стабильное и безотказное функционирование ИТ-инфраструктуры, осуществляет мониторинг, проводит инвентаризацию, отвечает за безопасность пользователей, занимается сетями и т.д. Это многорукий и многоголовый бог ИТ-инфраструктуры, который берёт на себя обязанности по обеспечению всей ИТ-жизнедеятельности компании. Встречается практически в любых компаниях.
  • Системный архитектор-инженер — специалист, проектирующий ИТ-инфраструктуру и архитектуру сети в крупных корпорациях.
  • Сетевой администратор — специалист, который занимается настройкой и развитием физических и логических сетей в компании, а также управлением системами биллинга, учёта и контроля трафика. Востребован в ЦОДах, телекоме, банках, корпорациях.
  • Инженер информационной безопасности — специалист, который обеспечивает безопасность ИТ-инфраструктуры на всех уровнях. Востребован в компаниях, чувствительных к атакам и проникновению в сеть (а это и финтех, и банки, и промышленность, и проч.).

Знание модели OSI

Модель OSI (The Open Systems Interconnection model) — это сетевая модель стека сетевых протоколов, разработанная ISO (Международной организацией по стандартизации). Она определяет семь уровней взаимодействия систем, позволяя различным сетевым устройствам взаимодействовать друг с другом.

  • Физический уровень: Определяет физические и электрические характеристики соединения.
  • Канальный уровень: Обеспечивает надежную передачу данных между узлами.
  • Сетевой уровень: Отвечает за маршрутизацию данных между устройствами.
  • Транспортный уровень: Обеспечивает надежную передачу данных между конечными узлами.
  • Сеансовый уровень: Управляет установкой, поддержкой и завершением сеансов связи.
  • Представительный уровень: Обеспечивает преобразование данных для приложений.
  • Прикладной уровень: Обеспечивает сетевые услуги для приложений пользователя.

Скриптинг на Bash

Bash — это командная оболочка Unix/Linux, которая позволяет автоматизировать задачи через скрипты. Вот некоторые ключевые моменты:

1.Создание скрипта:
Создайте текстовый файл с расширением .sh. Начните скрипт с #!/bin/bash, чтобы указать, что это Bash-скрипт. Напишите команды, используя синтаксис Bash.

2.Переменные и аргументы:
Используйте переменные для хранения значений. Передавайте аргументы в скрипт через $1, $2, и так далее.

3.Условия и циклы:
Используйте if, elif и else для условий. Циклы for и while помогут вам обрабатывать данные.

Скриптинг на PowerShell

PowerShell — это инструмент для автоматизации задач в Windows. Вот некоторые основные моменты:

1.Создание скрипта:
Создайте файл с расширением .ps1. Начните скрипт с #, чтобы указать, что это PowerShell-скрипт. Используйте команды, такие как Get-ChildItem, Set-Variable, и другие.

2.Модули и функции:
Модули содержат наборы функций и команд. Создавайте собственные функции для повторного использования кода.

3.Работа с объектами:
PowerShell работает с объектами, а не только с текстом. Используйте команды для обработки файлов, реестра и других ресурсов.

Скриптинг на Bash и PowerShell

Скриптинг - это написание программ (скриптов) для автоматизации задач. Bash и PowerShell - два популярных инструмента для скриптинга.

  • Bash: Это командный интерпретатор и язык сценариев для Unix-подобных систем. Bash позволяет автоматизировать задачи, управлять файлами и процессами, и выполнять сложные команды.
  • PowerShell: Это автоматизационный фреймворк и язык сценариев от Microsoft, используемый для управления и автоматизации задач в Windows. PowerShell интегрирован с .NET и позволяет выполнять команды управления системой.

Практические задания по скриптингу на Bash


Задание 1: Приветствие пользователя
Описание: Напишите Bash-скрипт, который приветствует пользователя по имени.
Требования:

Скрипт должен запрашивать имя пользователя.
Скрипт должен выводить приветственное сообщение с именем пользователя.
Пример скрипта: 

        #!/bin/bash

# Запрос имени пользователя
echo "Введите ваше имя:"
read user_name

# Приветственное сообщение
echo "Привет, $user_name!"
Задание 2: Проверка существования файла
Описание: Напишите Bash-скрипт, который проверяет, существует ли указанный файл.
Требования:

Скрипт должен запрашивать путь к файлу.
Скрипт должен проверять, существует ли файл, и выводить соответствующее сообщение.
Пример скрипта:

    #!/bin/bash

# Запрос пути к файлу
echo "Введите путь к файлу:"
read file_path

# Проверка существования файла
if [ -e "$file_path" ]; then
    echo "Файл $file_path существует."
else
    echo "Файл $file_path не существует."
fi

Практические задания по скриптингу на PowerShell

Задание 1: Приветствие пользователя
Описание: Напишите PowerShell-скрипт, который приветствует пользователя по имени.
Требования:

Скрипт должен запрашивать имя пользователя.
Скрипт должен выводить приветственное сообщение с именем пользователя.
Пример скрипта: 

    # Запрос имени пользователя
$userName = Read-Host "Введите ваше имя"

# Приветственное сообщение
Write-Host "Привет, $userName!"
Задание 2: Проверка существования файла
Описание: Напишите PowerShell-скрипт, который проверяет, существует ли указанный файл.
Требования:

Скрипт должен запрашивать путь к файлу.
Скрипт должен проверять, существует ли файл, и выводить соответствующее сообщение.
Пример скрипта: 

    # Запрос пути к файлу
$filePath = Read-Host "Введите путь к файлу"

# Проверка существования файла
if (Test-Path $filePath) {
    Write-Host "Файл $filePath существует."
} else {
    Write-Host "Файл $filePath не существует."
}

Ремонт компьютера

Системные администраторы часто занимаются диагностикой и ремонтом компьютерного оборудования. Это включает замену комплектующих, устранение неполадок и профилактическое обслуживание.

Обслуживание компьютера — важный аспект для его эффективной работы и продления срока службы. Вот несколько шагов, которые помогут вам поддерживать ваш ПК в хорошем состоянии: 1.Программное обеспечение и жесткий диск:

  • Очистите мусор, оставленный браузерами, используя утилиту “Очистка диска” в Windows или аналогичные программы.
  • Удалите программы-шпионы и вирусы.
  • Ежемесячно дефрагментируйте жесткий диск.
  • Удалите неиспользуемые программы и музыку.
  • Исключите из автозагрузки неиспользуемые программы.
  • Используйте утилиты для управления дисками.
2.Оборудование и процессор:
  • Включайте компьютер через источник бесперебойного питания (ИБП).
  • Регулярно удаляйте пыль из системного блока, чтобы предотвратить перегрев компонентов

Работа с настройкой и маршрутизацией компьютерных сетей

Настройка и маршрутизация сетей - это ключевая задача системного администратора. Она включает настройку маршрутизаторов, коммутаторов и других сетевых устройств, а также управление IP-адресацией и создание сетевых политик.

Маршрутизация в IP-сетях

Маршрутизация служит для приема пакета от одного устройства и передачи его по сети другому устройству через другие сети. Маршрутизаторы направляют трафик во все сети, составляющие объединенную сеть. Для маршрутизации пакета маршрутизатор должен владеть следующей информацией:

  • Адрес назначения: Куда направить пакет.
  • Соседний маршрутизатор: От которого он может узнать об удаленных сетях.
  • Доступные пути ко всем удаленным сетям.
  • Наилучший путь к каждой удаленной сети.

Методы обслуживания информации о маршрутизации

  • Статическая маршрутизация: Ввод администратором вручную местоположения всех сетей в таблицу маршрутизации.
  • Динамическая маршрутизация: Протоколы маршрутизации автоматически информируют об изменениях в сети.

Что такое маршрутизатор?

Маршрутизатор (шлюз) — узел сети с несколькими IP-интерфейсами, подключенными к разным IP-сетям. Он перенаправляет дейтаграммы из одной сети в другую для доставки от отправителя к получателю.

Установка офисных программ

Системные администраторы часто занимаются установкой и настройкой офисных программ, таких как Microsoft Office, для пользователей в организации. Это включает установку программного обеспечения, настройку учетных записей и предоставление технической поддержки.

Сетевой и инфраструктурный мониторинг

Мониторинг сетевой и инфраструктурной работы - это важная задача для обеспечения стабильности и безопасности сети. Системные администраторы используют инструменты мониторинга для отслеживания состояния сетевых устройств, серверов и служб, выявления проблем и предотвращения сбоев.

  • Инструменты мониторинга: Системные администраторы используют различные инструменты, такие как Nagios, Zabbix, SolarWinds, для мониторинга состояния сетевой инфраструктуры и серверов.
  • Анализ журналов: Анализ журналов событий помогает в выявлении и устранении проблем.
  • Проактивное обслуживание: Регулярное обслуживание и обновление оборудования и программного обеспечения позволяет предотвратить потенциальные сбои.
